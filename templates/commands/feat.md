# Feat - 兼容入口

$ARGUMENTS

## 执行流程

### 阶段 0：需求澄清（Brainstorming Pre-Pass）

在交给 `/ccg:manage` 前，先执行需求澄清：

1. **检查项目上下文**：读取项目 CLAUDE.md（若存在），理解技术栈与约束
2. **单问题逐步澄清**（每次只问一个问题，直到所有模糊点消除）：
   - 功能的核心目的是什么？
   - 成功标准（验收条件）是什么？
   - 有哪些已知约束（性能、兼容性、范围边界）？
3. **提出 2-3 种实现路径**（带推荐），等待用户选择或自动使用推荐方案
4. **构建结构化需求摘要**：
   ```
   目标：<一句话>
   技术路径：<选定方案>
   约束：<列表>
   验收标准：<列表>
   ```

### 阶段 1：移交 manage

将结构化需求摘要作为增强后的 $ARGUMENTS 传入：

```bash
/ccg:manage <结构化需求摘要>
```

若任务偏前端，附加 `frontend-focus`；偏后端附加 `backend-focus`。
